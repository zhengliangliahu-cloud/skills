## CNN 家族

### 任务特征

- 局部相关性强，平移不变性重要。
- 样本量有限，需要较强先验。
- 输入具有明确局部邻域，如图像、频谱图、局部时窗、脑电通道邻近结构。

### 适合的架构

- 2D/3D CNN、ResNet、DenseNet、Inception。
- 对时序和生物信号可用 1D CNN、深度可分离卷积、multi-branch temporal CNN。

### 优点

- 参数高效，局部模式建模稳定。
- 小中规模数据常比 Transformer 更稳。
- 训练成熟，部署友好，延迟低。

### 缺点

- 长程依赖和全局交互需要堆深层或配合注意力。
- 拓扑或跨通道关系不是原生优势。

### 算力/样本需求

- 对小样本更友好。
- 高分辨率输入会带来较高显存，但通常仍低于全局注意力。

### 常用技巧

- 残差、归一化、深度可分离卷积、多尺度卷积核。
- 先用 CNN 建局部表征，再接 attention / temporal module。

### 常见坑

- 以为卷积核变大就等于解决长程依赖。
- 对不规则图结构或跨远距离节点关系，CNN 往往不是首选。

### 何时不该用

- 序列特别长且全局依赖主导。
- 图结构或跨实体关系才是核心信息。

## RNN / LSTM / GRU

### 任务特征

- 顺序性强，在线更新重要。
- 序列较短到中等长度，或延迟敏感。

### 适合的架构

- Vanilla RNN 很少单独推荐。
- LSTM / GRU、双向 RNN、带 attention 的 RNN。

### 优点

- 对严格时序和在线推理自然。
- 小模型场景、低延迟场景仍有价值。

### 缺点

- 并行性差。
- 长序列梯度传播困难，虽然 LSTM/GRU 好于普通 RNN，但仍不如现代长序列路线。

### 常用技巧

- 梯度裁剪、层归一化、残差连接、截断 BPTT。
- 与 CNN 结合，先抽局部特征再建模序列。

### 何时不该用

- 序列超长、训练吞吐要求高、需要强上下文检索时。

## TCN 与卷积时序模型

### 任务特征

- 序列依赖明显，但仍希望保留卷积的并行和稳定性。
- 时间序列、工业信号、生物信号、传感器数据。

### 适合的架构

- TCN、dilated causal CNN、WaveNet 风格结构。

### 优点

- 并行训练，长感受野可通过 dilation 扩展。
- 常是 RNN 的强替代，在中等长度时序上很稳。

### 缺点

- 感受野设计需要任务驱动。
- 对跨段全局交互和内容寻址不如 Transformer 灵活。

### 常见坑

- 只调 dilation 不看任务时间尺度。
- 窗口与标签对齐错误会直接污染结论。

## Transformer 家族

### 任务特征

- 长程依赖、全局交互、多头关系建模重要。
- 数据量较大，或能借助预训练。

### 适合的架构

- Encoder-only、Decoder-only、Encoder-Decoder。
- ViT、Time Series Transformer、Conformer、Longformer、Performer 等高效变体。

### 优点

- 全局依赖强，扩展到多模态和大规模预训练方便。
- 容易接入自监督、检索、参数高效微调。

### 缺点

- 数据饥饿，易在小样本和高噪声任务中过拟合。
- 全局注意力在长序列上成本高。

### 算力/样本需求

- 通常比 CNN / TCN 更吃数据和显存。
- 小样本时往往需要强正则、预训练或混合结构。

### 常用技巧

- 位置编码、层缩放、warmup、label smoothing、drop path。
- 对局部结构强的任务，用 CNN / patch embedding / local attention 先注入先验。

### 何时不该用

- 样本太少且无可迁移预训练。
- 真正的瓶颈是评估协议、数据清洗或域偏移，而非上下文窗口。

## GNN 家族

### 任务特征

- 节点、边、拓扑关系才是核心。
- 任务依赖邻居聚合、消息传播、多跳结构。

### 适合的架构

- GCN：适合平滑、同配图、基础节点分类。
- GraphSAGE：适合大图采样和归纳设置。
- GAT：适合邻居重要性不均、异质邻域。
- MPNN / GIN：图级任务、分子图常见。
- Graph Transformer / Graphormer：需要更强全局交互和结构编码时。

### 优点

- 天然编码关系结构。
- 在分子、推荐、知识图谱、脑网络建模中比序列/卷积更贴题。

### 缺点

- 过平滑、过压缩、采样偏差、图构造误差都很常见。
- 图不是银弹；图构造错了，GNN 往往稳定放大噪声。

### 常见坑

- 先验图构造缺乏任务依据。
- 把动态关系强行静态化。
- 在本质不是图问题时硬上 GNN。

### 何时不该用

- 拓扑只是弱信息，主信息仍在原始连续信号或文本上下文里。
- 图构造不可靠且难验证。

## SSM / Mamba / 线性递归路线

### 任务特征

- 长序列、高吞吐、低延迟、长上下文效率是核心。
- 希望保留递归或卷积路线的线性复杂度，同时改善内容选择能力。

### 适合的架构

- S4、S5、Mamba、混合 Transformer-SSM 路线。

### 优点

- 对超长序列更省内存和更高吞吐。
- 在部分语音、基因组、长文本、长时间序列任务上有明显吸引力。

### 缺点

- 生态和经验积累不如 Transformer 完整。
- 任务若强依赖显式任意位置交互，纯 SSM 方案不一定占优。

### 常见坑

- 把“线性复杂度”误当“必然更好”。
- 忽视实现成熟度、kernel 支持和训练稳定性。

## MoE、Autoencoder、Diffusion 与混合结构

### MoE

- 适合大规模容量扩展和条件计算。
- 主要问题是路由不均、训练复杂和推理系统要求高。

### Autoencoder / VAE

- 适合表征学习、降维、异常检测、生成式潜变量建模。
- 单独做判别任务时，未必比直接监督架构更强。

### Diffusion

- 适合高质量生成、去噪、逆问题和部分时序生成。
- 训练与采样成本高，不是通用判别 backbone。

### 混合结构

- CNN + Transformer：局部先验 + 全局交互。
- CNN/TCN + GNN：原始信号 + 关系图。
- Transformer + retrieval / memory：内容检索增强。
- Transformer + SSM：折中全局表达与长序列效率。

## 快速选型规则

- 局部纹理/局部时频结构主导：先看 CNN。
- 序列中等长度且在线推理重要：先看 GRU/LSTM/TCN。
- 长程依赖和全局交互主导，且数据够大：先看 Transformer 或混合结构。
- 拓扑关系是任务本体：先看 GNN / Graph Transformer。
- 超长序列和吞吐/显存压力主导：考虑 SSM/Mamba 或高效 Transformer。
- 小样本高噪声：优先强先验架构和训练协议，不要先押最前沿 backbone。
