## 建模套路与常见反模式

### 工厂排产

- 首选：产销协同做 MILP，细排程做 `CP-SAT` 或混合框架。
- 先拆主计划与细排程，不要一步到位做全年分钟级排程。
- 常见反模式：切换、停机、工装约束后补，导致主模型返工。

### 车辆路径与配送

- 首选：图算法或 `OR-Tools Routing` 造初解，复杂约束时升到 set partitioning / branch-and-price / ALNS。
- 先定义客户服务规则、时间窗、载重、班次和回场策略。
- 常见反模式：所有配送细则都堆进第一次建模。

### 人员排班

- 首选：`CP-SAT` 或 set partitioning。
- 先区分硬法规、软偏好、技能匹配、稳定性目标。
- 常见反模式：不处理同质班次对称性。

### 设施选址与网络设计

- 首选：固定费用选址、覆盖、p-median、网络流耦合。
- 先做候选点筛选、需求聚合、半径预过滤。
- 常见反模式：候选点过多却没有预处理。

### 库存补货

- 首选：确定型先做 EOQ / base-stock / LP，随机性强时再上鲁棒或随机规划。
- 先确认补货周期、提前期、服务水平和缺货成本。
- 常见反模式：需求噪声很大却用过度精细的静态优化。

### 资源分配与产能规划

- 首选：LP / MILP，必要时做滚动重规划。
- 先确认能力边界、合同约束、切换损失、最低服务承诺。
- 常见反模式：把长期配置和短期调度混为一体。

## 从初级到高级的算法谱系

- 初级：规则法、图算法、LP baseline。
- 中级：MILP、CP-SAT、Routing、rolling horizon。
- 进阶：鲁棒/随机规划、Benders、列生成、LNS/ALNS。
- 高级：matheuristic、branch-and-price、主从混合工业框架。

## 参数调优清单

- 排产：先调时间粒度、换型建模、warm start，再调 MIP/CP 参数。
- 配送：先调候选邻居、初解质量、LNS 邻域，再调搜索深度。
- 排班：先调对称性破坏、关键员工优先级、可行性搜索阶段。
- 选址：先调候选点过滤、需求聚合、有效上界。
- 库存：先调服务水平、场景数、滚动窗口。
- 资源规划：先调冻结窗口、重规划频率、稳定性罚项。

## 优缺点与失效模式

高频失败模式：

- 两小时无可行解：先查数据矛盾、变量域、`big-M`、硬约束冲突，再提高启发式或喂 warm start。
- 有可行解但很差：先做目标归一化、主约束强化、关键变量优先分支，再换求解器参数。
- gap 不降：先查 LP 松弛是否过弱、是否有对称性、是否该换分解或 set partitioning。
- 结果不稳定：先引入方案稳定性目标、历史解继承、滚动冻结窗口。
- 线上反复失效：先补 fallback heuristic、可行性修复器、失败回退规则。

## 切换与升级信号

- 工厂排产在 MILP 与 `CP-SAT` 之间犹豫时：成本与资源平衡主导用 MILP，时序与逻辑主导用 `CP-SAT`，混合时拆层。
- 车辆路径开源路线失效时：若 Routing + ALNS 仍达不到质量或时效，再看 set partitioning + 商业 MIP。
- 需求不确定明显时：从确定型补货切到 `stochastic-robust-online-and-multiobjective.md`。
- 精确法停滞时：转到 `metaheuristics-and-matheuristics.md` 设计 fix-and-optimize、ALNS 或 ruin-and-recreate 兜底。
